THIET KE HE THONG PAYMENT BANG USDC

Muc tieu

Them 1 phuong thuc nap tien moi bang USDC vao he thong wallet hien tai, bao gom:

1. User tao yeu cau nap tien tren frontend
2. He thong tra ve so USDC chinh xac can chuyen + dia chi nhan + QR
3. Backend tu dong theo doi blockchain, xac nhan giao dich, cong tien vao vi noi bo
4. Admin co man hinh review cac case lech so tien / sai token / can xu ly thu cong
5. Toan bo flow phai ghep duoc vao stack hien co: NestJS + TypeORM + BullMQ + Next.js + React Query + Socket

Quyet dinh kien truc cho phase 1

Khong nen dung "moi deposit = 1 vi EVM rieng" ngay tu dau, vi se phat sinh bai toan sweep USDC va cap gas native token cho tung vi con.

Phuong an de ship nhanh, an toan va phu hop codebase hien tai:

1. Chi ho tro 1 chain duy nhat trong phase 1
   De xuat dat ten generic la `usdc`, nhung config chi tro den 1 network duy nhat, vi du `base` hoac `polygon`.

2. Dung 1 treasury address co dinh
   User chuyen USDC vao 1 dia chi he thong duy nhat.

3. Match giao dich bang "so USDC chinh xac"
   Tuong tu noi dung chuyen khoan cua bank QR, he thong tao 1 quote USDC exact cho moi deposit.
   Vi du:
   - User muon nap 100.000d
   - Rate khoa tai thoi diem tao deposit = 25.000 VND / 1 USDC
   - So USDC co ban = 4.000000
   - He thong gan suffix duy nhat -> 4.000137 USDC
   - User phai chuyen dung 4.000137 USDC vao treasury address

4. Vi noi bo van giu don vi VND nhu hien tai
   `deposits.amount` tiep tuc la so tien cong vao vi noi bo.
   USDC chi la payment rail, khong doi toan bo wallet sang crypto.

5. Auto-credit chi khi match exact amount + dung token + dung network + du confirmations
   Moi truong hop sai lech dua vao `review_required` de admin xu ly.

6. Khong bat user connect wallet o phase 1
   Frontend chi can hien:
   - so USDC can chuyen
   - network
   - treasury address
   - QR / deep link
   - trang thai on-chain
   - nut "Kiem tra lai" neu user muon refresh som

7. Can admin fallback
   Vi giao dich on-chain co the gap underpayment, overpayment, tx den sau khi quote het han, gui sai token, hoac watcher bo lo.

3. Migration de them cot cho USDC

ALTER TABLE deposits
  ADD COLUMN receiver_address VARCHAR(80),
  ADD COLUMN sender_address VARCHAR(80),
  ADD COLUMN network_code VARCHAR(20),
  ADD COLUMN asset_symbol VARCHAR(20),
  ADD COLUMN asset_decimals INTEGER,
  ADD COLUMN token_contract VARCHAR(80),
  ADD COLUMN quoted_asset_amount_atomic BIGINT,
  ADD COLUMN received_asset_amount_atomic BIGINT,
  ADD COLUMN quote_rate_vnd BIGINT,
  ADD COLUMN tx_hash VARCHAR(100),
  ADD COLUMN tx_nonce BIGINT,
  ADD COLUMN block_number BIGINT,
  ADD COLUMN confirmations INTEGER DEFAULT 0,
  ADD COLUMN review_reason VARCHAR(50),
  ADD COLUMN detected_at TIMESTAMPTZ;

CREATE INDEX idx_deposits_usdc_status
  ON deposits(payment_method, status, created_at DESC);

CREATE INDEX idx_deposits_usdc_tx_hash
  ON deposits(tx_hash)
  WHERE tx_hash IS NOT NULL;

CREATE INDEX idx_deposits_usdc_quote
  ON deposits(payment_method, network_code, quoted_asset_amount_atomic)
  WHERE payment_method = 'usdc';

Co the them unique partial index cho active quotes:

CREATE UNIQUE INDEX idx_deposits_usdc_active_unique_quote
  ON deposits(payment_method, network_code, quoted_asset_amount_atomic)
  WHERE payment_method = 'usdc'
    AND status IN ('pending', 'detected', 'confirming');

4. Bang cursor de scan blockchain

Them bang nho de luu block da quet:

CREATE TABLE payment_chain_cursors (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  provider VARCHAR(30) NOT NULL,          -- 'usdc'
  network_code VARCHAR(20) NOT NULL,      -- 'base'
  token_contract VARCHAR(80) NOT NULL,
  last_scanned_block BIGINT NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (provider, network_code, token_contract)
);

Ly do:

- watcher khong phai scan lai tu block 0
- de retry an toan neu service restart

5. Quy uoc du lieu

- `amount`: so tien cong vao wallet noi bo (VND)
- `quoted_asset_amount_atomic`: so USDC user phai gui, don vi atomic 10^-6
- `received_asset_amount_atomic`: so USDC he thong thuc nhan tren chain
- `quote_rate_vnd`: rate da khoa tai luc tao deposit

Tuyet doi khong dung float cho USDC o backend.

3. Quote service

`UsdcQuoteService` chiu trach nhiem:

1. Nhan `amount` VND
2. Lay `quoteRateVnd`
3. Tinh `baseAtomicAmount`
4. Gan suffix unique de tranh trung quote dang active
5. Tra ve:
   - `creditedAmountVnd`
   - `quotedAssetAmountAtomic`
   - `quotedAssetDisplay`
   - `rateVnd`

Pseudo:

```ts
const baseAtomicAmount = Math.ceil((amountVnd * 1_000_000) / rateVnd);
const uniqueSuffix = await allocateUniqueSuffix(); // vi du 137
const quotedAtomicAmount = baseAtomicAmount + uniqueSuffix;
```

Rule:

- Chi uniqueness trong tap active deposits
- Neu suffix clash thi retry
- Khong duoc lam quoted amount vuot max business rule

Neu muon an toan, service tra ca:

5. Tao deposit USDC trong `PaymentService`

Cap nhat `createDeposit()`:

```ts
switch (paymentMethod) {
  case 'usdc':
    return this.createUsdcDeposit(userId, dto.amount);
  case 'momo_qr':
  case 'vcb_qr':
  case 'ocb_qr':

```

`createUsdcDeposit()`:

3. expire deposit qua han cua user neu co
4. neu user da co 1 USDC deposit active thi tra ve ban ghi do thay vi tao moi
5. tao quote USDC exact
6. tao QR + deep link
7. luu deposit:

   - `amount = VND credited`
   - `receiverAddress = treasury address`
   - `networkCode`
   - `assetSymbol = 'USDC'`

   - `tokenContract`

   - `quoteRateVnd`
   - `status = 'pending'`
   - `expiresAt`
8. enqueue job expire deposit nhu flow hien tai
9. return deposit + payment info cho frontend

7. Watcher service

`UsdcWatcherService` co 2 cach:

1. Polling RPC bang HTTP
2. Subscription bang WebSocket

Cho repo nay, uu tien polling + BullMQ de de van hanh:

- them job `sync-usdc-transfers`
- repeat moi 15-30 giay
- luu `last_scanned_block` trong `payment_chain_cursors`
- doc `Transfer` logs cua token contract
- filter `to == treasuryAddress`
- tim deposit active co `quoted_asset_amount_atomic == value`

Implementation hint:

- dung `viem` public client
- query `getLogs` theo ABI event `Transfer(address indexed from, address indexed to, uint256 value)`
- sau do goi `getBlockNumber()` de tinh confirmations

Flow matching:

1. Lay block range chua scan
2. Query logs token `Transfer(from, to, value)`
3. Bo qua log khong dung treasury address
4. Match bang:
   - `paymentMethod = 'usdc'`
   - `networkCode` khop
   - `quotedAssetAmountAtomic = value`
   - status nam trong `pending | detected | confirming`
   - `expiresAt` chua qua han hoac co chinh sach grace window
5. Neu khong match -> bo qua
6. Neu match:
   - luu `txHash`
   - `senderAddress`
   - `receivedAssetAmountAtomic`
   - `blockNumber`
   - `detectedAt`
   - tinh confirmations
7. Neu confirmations < required -> `confirming`
8. Neu confirmations >= required -> goi `completeUsdcDeposit()`

8. Complete deposit on-chain

`completeUsdcDeposit()` phai chay trong transaction DB:

1. lock deposit row
2. verify lai:
   - payment method = usdc
   - status chua completed
   - amount on-chain = quoted amount
   - token contract dung
3. lock wallet row
4. cong `wallet.balance += deposit.amount`
5. update deposit:
   - `status = 'completed'`
   - `completedAt = now`
   - `confirmations = ...`
   - `webhookData` / raw on-chain metadata
6. tao `transactions` row:
   - `type = 'deposit'`
   - `status = 'completed'`
   - `referenceType = 'deposit'`
   - `metadata.provider = 'usdc'`
   - `metadata.network = networkCode`
   - `metadata.txHash = txHash`
   - `metadata.quotedAssetAmountAtomic`
   - `metadata.receivedAssetAmountAtomic`
   - `metadata.quoteRateVnd`

Sau transaction DB:

- emit `wallet:balance_updated`
- tao notification "Nap tien USDC thanh cong"

9. Xu ly lech so tien / sai du lieu

Rule de xuat cho phase 1:

- Exact amount: auto complete
- Nho hon quote: `review_required`, reason = `underpaid`
- Lon hon quote: `review_required`, reason = `overpaid`
- Dung amount nhung quote da het han: `review_required`, reason = `late_payment`
- Sai token contract: watcher bo qua, khong auto hoan tien
- Sai network: he thong khong detect duoc, UI phai canh bao rat ro
- 1 quote da co txHash roi ma lai nhan tx hash thu 2: `review_required`, reason = `duplicate_payment`

Khong nen auto credit partial trong phase 1.
Admin moi la nguoi quyet dinh case ngoai le.

11. Admin review logic

Can support 3 action:

1. Approve exact quote
   Credit dung `deposit.amount`

2. Approve custom VND amount
   Dung cho underpaid / overpaid / case can dieu chinh tay

3. Reject
   Khong cong vi, luu note ro ly do

Khuyen nghi:

- chi cho `creditedAmount` khi deposit.status = `review_required`
- luu `adminConfirmedBy`, `adminNote`, `completedAt`

12. Notification + realtime

Khi deposit USDC complete:

- emit socket `wallet:balance_updated`
- dispatch notification type moi, vi du `deposit_completed`

Neu chua muon mo rong notification type, it nhat emit event balance de frontend tu refresh.

13. Security va van hanh

Bat buoc:

1. So sanh token contract theo config, khong tin data tu frontend
2. So sanh chain/network theo config
3. Khong credit o 0 confirmation
4. Chot exact atomic amount, khong compare float
5. Pessimistic lock khi complete deposit
6. Job idempotent: scan lai nhieu lan khong duoc credit 2 lan
7. Validate treasury address va token contract tu env ngay luc boot app
8. Log day du txHash, blockNumber, senderAddress de audit
