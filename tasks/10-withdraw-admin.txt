 Chi tiết thực hiện

 1. Bug fix backend — rejectWithdrawal không lưu adminId

 File: backend/src/modules/admin/admin.service.ts

 Hiện tại method rejectWithdrawal nhận adminId nhưng không dùng:
 async rejectWithdrawal(withdrawalId: string, adminId: string) {
   // ... adminId bị bỏ qua
   withdrawal.status = 'rejected';
   withdrawal.approvedBy = adminId; // ← BUG: dòng này THIẾU

 Sửa: Thêm withdrawal.approvedBy = adminId; và withdrawal.completedAt = new Date(); như bên approveWithdrawal.

 2. Filter bar client-side — Frontend

 Thêm state và UI filter vào AdminWithdrawalsPanel:

 State cần thêm:

 const [searchText, setSearchText] = useState('');          // tìm theo tên, email, SĐT, STK
 const [statusFilter, setStatusFilter] = useState<'all' | WithdrawalStatus>('all');

 Logic filter (useMemo):

 const filteredItems = useMemo(() => {
   return items.filter(item => {
     // Filter theo status
     if (statusFilter !== 'all' && item.status !== statusFilter) return false;
     // Filter theo search text (case-insensitive)
     if (searchText.trim()) {
       const q = searchText.toLowerCase();
       const matchName = item.user?.displayName?.toLowerCase().includes(q);
       const matchEmail = item.user?.email?.toLowerCase().includes(q);
       const matchBank = item.bankAccount?.toLowerCase().includes(q);
       const matchHolder = item.bankHolder?.toLowerCase().includes(q);
       if (!matchName && !matchEmail && !matchBank && !matchHolder) return false;
     }
     return true;
   });
 }, [items, searchText, statusFilter]);

 UI filter bar (thêm phía trên danh sách):

 - Input tìm kiếm (placeholder: "Tìm theo tên, email, số tài khoản...")
 - Dropdown hoặc SegmentedControl chọn trạng thái: Tất cả / Chờ duyệt / Hoàn tất / Từ chối
 - Hiển thị số kết quả: "Đang hiển thị X / Y withdrawal"
 - Dùng components sẵn: Input từ @/components/ui/input, Select từ @/components/ui/select

 Tách pending/processed từ filteredItems (không phải items):

 const pendingItems = filteredItems.filter(item => item.status === 'pending');
 const processedItems = filteredItems.filter(item => item.status !== 'pending');

 3. Confirm dialog trước khi approve/reject

 Dùng AlertDialog từ @/components/ui/alert-dialog (shadcn/ui đã có sẵn):

 // State trong AdminWithdrawalsPanel
 const [confirmState, setConfirmState] = useState<{
   open: boolean;
   withdrawalId: string;
   action: 'approve' | 'reject';
   amount: number;
   userName: string;
 } | null>(null);

 Flow:
 1. User click "Duyệt và hoàn tất" → setConfirmState mở dialog
 2. Dialog hiển thị: "Xác nhận duyệt rút 100,000đ cho Nguyễn Văn A?"
 3. User click "Xác nhận" → gọi mutation
 4. Dialog đóng

 WithdrawalCard nhận onApprove và onReject props giữ nguyên — chỉ đổi implementation trong AdminWithdrawalsPanel để mở dialog thay vì gọi mutation trực tiếp.
